# yolo26
利用yolo26进行深度学习的端到端模型的训练
该模型训练出来的全新的权重文件相较于yolov5的识别速度变慢了，但是置信度提升了大概10%
需注意本文件中的compare文件为自创用于比较nano版本的不同模型，以及根目录下的train文件是基于windos系统写的，如果要换成Linux系统需要去除文件的入口并重新更改路径
训练过程如下：
使用make_sense对识别物体打标签，其中需注意标签形式输出为txt文件（该txt文件要和jpg文件一一对应）
找到dataset下的my_dataset文件，并在此目录下创建两个文件夹分别是image用于存放jpg格式的文件以及labels用来存放txt格式的文件
在images与labels中都创建train文件夹和val文件夹这两个文件的内容为相应的上个文件夹内容（要注意训练集和验证集的数据比为8：2）
在根目录Ultratics文件中找到cfg文件下的datasat中的VOC文件（后缀名为yaml）
在上一步的VOC文件中操作：
{第12行的path文件改为dataset的路径（需为绝对路径）
在文件的标签部分改为自己的标签名}
在根目录下的train.py文件中训练模型
{在该文件中需要改权重模型的存储位置和训练所使用的标准模型}
接下来的部分是推理，在根目录中找到predict.py文件
{在该文件的第3行改为自己的权重文件的绝对路径
在第4行course中将路径改为自己想要验证的
在第7行的project文件修改保存的位置}
最后是验证的部分，在根目录下找到eval.py
{在第3行修改权重文件的位置
在第4行修改VOC.yaml文件的位置}
#需注意的是该模式训练的yolo26模型为nano版本，并且相较于Ultratics官方的文件添加了train文件与compare文件
